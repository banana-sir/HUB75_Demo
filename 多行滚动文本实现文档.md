## 概述

多行滚动文本功能允许在HUB75 LED矩阵上同时显示多行不同字体大小的滚动文本。通过将每行的信息独立存储，实现了不同字体大小的文本互不干扰地同时滚动。

### 关键特性
- ✅ 支持多行同时滚动（最多PANEL_RES_Y/16行）
- ✅ 每行独立的字体大小
- ✅ 每行独立的滚动速度
- ✅ 每行独立的滚动方向
- ✅ 每行独立的文本颜色
- ✅ 自动检测并清除重叠文本
- ✅ 动态适配不同屏幕分辨率
- ✅ 50fps 高刷新率（DMA双缓冲）

## 核心设计

### ScrollLine结构体

每行滚动文本由`ScrollLine`结构体独立管理：

```cpp
struct ScrollLine {
    char *content;               // 滚动文本内容（动态分配）
    int xPosition;               // 当前X坐标位置
    int yPosition;               // 行的Y坐标（固定）
    bool isScrolling;            // 是否正在滚动
    bool isActive;               // 该行是否激活
    int updateFrameInterval;     // 该行的更新时间间隔（毫秒）
    unsigned long lastUpdateTime;  // 该行上次更新的时间戳
    ScrollDirection scrollDirection;  // 滚动方向
    uint16_t color;              // 该行的文本颜色
    int textSize;                // 该行的字体大小（独立记录）
    uint16_t cachedTextWidth;    // 缓存的文本宽度（避免重复计算）
};
```

**关键设计要点：**
1. **动态内存**：content使用动态分配，支持灵活的文本长度
2. **独立状态**：每行维护自己的滚动状态，互不干扰
3. **时间控制**：每行独立的时间间隔，实现不同速度
4. **缓存优化**：cachedTextWidth避免每帧重复计算文本宽度

### 滚动方向枚举

```cpp
enum ScrollDirection {
    SCROLL_LEFT = 0,   // 向左滚动
    SCROLL_RIGHT = 1   // 向右滚动
};
```

## Y坐标系统

### Y坐标计算规则

滚动文本的Y坐标固定为 `(line - 1) * 16`，不随字体大小变化：

| 行号(line) | Y坐标 | 说明 |
|-----------|-------|------|
| 1 | 0 | 第1行，从y=0开始 |
| 2 | 16 | 第2行，从y=16开始 |
| 3 | 32 | 第3行（64x64屏幕）|
| 4 | 48 | 第4行（64x64屏幕）|

### 设计原因

1. **简化坐标系统**：每行有固定的起始Y坐标，便于计算和重叠检测
2. **支持多字体共存**：不同字体大小的文本可以共享同一行系统
3. **统一渲染逻辑**：在loop()中动态切换字体大小渲染不同行
4. **易于扩展**：修改屏幕高度时，最大行数自动计算

### 代码实现

```cpp
// DisplayManager.cpp: displayText()
int yPosition = (line - 1) * 16;
scrollLines[index].yPosition = yPosition;
```

### 字体高度与覆盖关系

不同字体大小的高度：

| 字体大小 | 高度（像素）| 占用行数 | 覆盖行号 |
|---------|------------|----------|----------|
| 1 | 16 | 1 | line |
| 2 | 32 | 2 | line, line+1 |

**示例：**
- `line=2, textSize=1`: 渲染区域 y=16~32
- `line=2, textSize=2`: 渲染区域 y=16~48，会覆盖line=3（如果存在）

## 字体大小共存

### 问题场景

如果先设置字体大小为1显示滚动文本，再设置字体大小为2显示另一行滚动文本，如果不做特殊处理，第一行文本会因为全局textSize改变而显示错误。

### 解决方案

#### 1. 每行独立记录字体大小

```cpp
struct ScrollLine {
    int textSize;  // 该行的字体大小（独立记录）
    // ...
};
```

#### 2. 创建滚动文本时记录字体大小

```cpp
void DisplayManager::displayText(...) {
    // ...
    // 设置该行的字体大小（使用当前的全局textSize）
    scrollLines[index].textSize = textSize;
    // ...
}
```

#### 3. 渲染时动态切换字体大小

```cpp
void DisplayManager::renderScrollingTexts() {
    for (int i = 0; i < maxLines; i++) {
        if (scrollLines[i].isActive && scrollLines[i].isScrolling && scrollLines[i].content) {
            // 设置该行的字体大小
            dma_display->setTextSize(scrollLines[i].textSize);
            dma_display->setTextColor(scrollLines[i].color);
            dma_display->setCursor(scrollLines[i].xPosition, scrollLines[i].yPosition);
            dma_display->printUTF8(scrollLines[i].content);
        }
    }

    // 恢复全局字体大小
    dma_display->setTextSize(textSize);
}
```

### 关键点

- `setTextSize()` 只改变全局 `textSize` 变量，不影响已存在的滚动文本
- 新的滚动文本使用当前的 `textSize` 并保存到 `ScrollLine.textSize`
- 渲染时使用每行自己的 `textSize`，实现多字体同时显示
- 恢复全局textSize，确保下次调用时使用正确的字体大小

## 重叠检测机制

### 静态文本覆盖滚动文本

当显示静态文本时，自动检测并清除被覆盖的滚动文本：

```cpp
void DisplayManager::displayText(...) {
    if (!isScroll) {
        // 计算静态文本的渲染区域
        int yPosition = (line - 1) * 16;
        int newHeight = requiredLines * 16;

        // 清除重叠的滚动文本
        clearOverlappingScrollTexts(yPosition, newHeight);
        // 清除重叠的静态文本
        clearOverlappingStaticTexts(yPosition, newHeight);
    }
}
```

### 滚动文本互相覆盖

当添加新滚动文本时，自动检测并清除被覆盖的其他滚动文本：

```cpp
void DisplayManager::displayText(...) {
    if (isScroll) {
        int yPosition = (line - 1) * 16;
        int lineHeight = textSize * 16;

        // 清除被新滚动文本覆盖的其他滚动文本
        clearOverlappingScrollTexts(yPosition, lineHeight, index);
        // 清除重叠的静态文本
        clearOverlappingStaticTexts(yPosition, lineHeight, index);
    }
}
```

### 重叠判断逻辑

两个矩形区域 `[y1, y1+h1)` 和 `[y2, y2+h2)` 重叠的条件：

```cpp
bool isOverlap(int y1, int h1, int y2, int h2) {
    return !(y1 + h1 <= y2 || y1 >= y2 + h2);
}
```

**解释：**
- 不重叠：区域1在区域2上方，或区域1在区域2下方
- 重叠：其他所有情况

```cpp
// DisplayManager.cpp: clearScrollLineByIndex()
void DisplayManager::clearScrollLineByIndex(int index) {
    if (!isValidScrollIndex(index)) return;

    if (scrollLines[index].content != nullptr) {
        free(scrollLines[index].content);
        scrollLines[index].content = nullptr;
    }
    scrollLines[index].isActive = false;
    scrollLines[index].isScrolling = false;
    scrollLines[index].cachedTextWidth = 0;
    scrollLines[index].lastUpdateTime = 0;
    scrollLines[index].xPosition = PANEL_RES_X;
}

// DisplayManager.cpp: freeAllScrollLines()
void DisplayManager::freeAllScrollLines() {
    if (scrollLines) {
        for (int i = 0; i < maxLines; i++) {
            if (scrollLines[i].content != nullptr) {
                free(scrollLines[i].content);
                scrollLines[i].content = nullptr;
            }
            scrollLines[i].isActive = false;
            scrollLines[i].isScrolling = false;
            scrollLines[i].cachedTextWidth = 0;
            scrollLines[i].lastUpdateTime = 0;
            scrollLines[i].xPosition = PANEL_RES_X;
        }
    }
}
```

- 在分配新文本前先释放旧文本
- 重叠检测时释放被覆盖的文本
- 全局清屏时释放所有滚动文本
- 析构函数中释放所有内存
- 使用 `nullptr` 检查避免重复释放

## 渲染流程

### DMA双缓冲机制

HUB75 LED矩阵显示采用DMA（Direct Memory Access）双缓冲机制，这是实现高刷新率和无闪烁显示的核心技术。

#### 双缓冲工作原理

```
┌─────────────────────────────────────────────────────────────┐
│                    DMA双缓冲系统                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌──────────────┐         ┌──────────────┐                 │
│  │ 前缓冲区     │         │ 后缓冲区     │                 │
│  │ (Front Buffer)│       │ (Back Buffer) │                 │
│  │              │         │              │                 │
│  │  当前显示中   │         │  正在绘制中   │                 │
│  │              │         │              │                 │
│  └──────┬───────┘         └──────┬───────┘                 │
│         │                        │                          │
│         │ DMA持续输出             │                          │
│         │ 到LED矩阵              │                          │
│         │                        │                          │
│         │                        │ 渲染操作                 │
│         │                        │ (clearScreen/draw)     │
│         │                        │                          │
│         │         flipDMABuffer() │                          │
│         └────────────────────────┘                          │
│                  交换缓冲区                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

#### 关键特性

1. **前缓冲区（Front Buffer）**
   - DMA硬件持续读取此缓冲区数据
   - 驱动LED矩阵显示
   - 渲染过程中不会被修改
   - 保证显示稳定，无闪烁

2. **后缓冲区（Back Buffer）**
   - 渲染操作在此缓冲区执行
   - 可以随时修改（清屏、绘制等）
   - 不会影响当前显示内容
   - 完成绘制后与前缓冲区交换

3. **flipDMABuffer()**
   - 原子操作，交换两个缓冲区指针
   - 由硬件触发，确保同步
   - 实际显示更新时机由DMA控制器决定

### 渲染时序与帧控制

#### 帧时间控制

```cpp
// DisplayManager.h
unsigned long lastFrameTime;  // 上一帧的时间戳
unsigned long frameInterval;  // 目标帧间隔（毫秒），默认20ms = 50fps
```

#### 主循环时序

```
时间轴：0ms    20ms   40ms   60ms   80ms   100ms
        │      │      │      │      │       │
        ▼      ▼      ▼      ▼      ▼       ▼
        ┌─────────────────────────────────────┐
        │ frameInterval = 20ms (50fps)       │
        └─────────────────────────────────────┘

Frame 1: 0ms ~ 20ms
  ├── updateScrolling()  更新滚动位置
  └── renderFrame()      渲染并flip
      ├── clearScreen()      清除后缓冲区
      ├── renderStaticTexts()   绘制静态文本
      ├── renderScrollingTexts() 绘制滚动文本
      └── flipDMABuffer()    交换缓冲区

Frame 2: 20ms ~ 40ms
  ├── updateScrolling()
  └── renderFrame()
      ...

Frame 3: 40ms ~ 60ms
  ...
```

#### 代码实现

```cpp
void DisplayManager::loop() {
    // 如果是图片显示模式，跳过渲染循环
    if (isImageDisplayMode) {
        return;
    }

    unsigned long now = millis();

    // 只在达到目标帧间隔时才进行更新和渲染（保持同步）
    if (now - lastFrameTime >= frameInterval) {
        updateScrolling();  // 先更新滚动位置
        renderFrame();      // 再渲染并flip
        lastFrameTime = now;
    }
}
```

### 静态文本与滚动文本共存机制

#### 共存策略

静态文本和滚动文本可以在同一帧中并存，通过渲染顺序和层次管理实现：

```
渲染层次（从底到顶）：

Layer 0: 背景层
  └─ clearScreen() 填充黑色

Layer 1: 静态文本层
  └─ renderStaticTexts() 绘制静态文本

Layer 2: 滚动文本层
  └─ renderScrollingTexts() 绘制滚动文本
     └─ 每行滚动文本前先清除该行区域
```

#### 关键设计要点

1. **先静态后滚动**
   - 静态文本先绘制到后缓冲区
   - 滚动文本后绘制，可以覆盖静态文本
   - 确保滚动文本优先显示

2. **滚动文本行清除**
   - 每行滚动文本绘制前先清除该行区域
   - 防止与静态文本混合显示
   - 确保滚动文本独立显示

3. **重叠自动处理**
   - 显示新文本时自动检测重叠
   - 清除被覆盖的旧文本
   - 避免视觉混乱

### 更新滚动状态

```cpp
void DisplayManager::updateScrolling() {
    if (!isDisplayReady() || scrollLines == nullptr) {
        return;
    }

    unsigned long now = millis();

    // 更新所有激活的滚动行，每行独立更新
    for (int i = 0; i < maxLines; i++) {
        if (scrollLines[i].isActive && scrollLines[i].isScrolling && scrollLines[i].content) {
            // 使用lastUpdateTime跟踪该行上次更新的时间
            if (!scrollLines[i].lastUpdateTime) {
                scrollLines[i].lastUpdateTime = now;
            }

            if (now - scrollLines[i].lastUpdateTime >= scrollLines[i].updateFrameInterval) {
                scrollLines[i].lastUpdateTime = now;

                // 更新文本位置（根据方向移动1像素）
                if (scrollLines[i].scrollDirection == SCROLL_LEFT) {
                    scrollLines[i].xPosition += SCROLL_OFFSET_LEFT;  // -1
                } else {
                    scrollLines[i].xPosition += SCROLL_OFFSET_RIGHT;  // +1
                }

                // 检查文本是否完全移出屏幕，根据滚动方向重置位置
                // 使用缓存的文本宽度，避免每帧调用getTextBounds()
                if (scrollLines[i].scrollDirection == SCROLL_LEFT) {
                    // 向左滚动：文本完全移出左侧时重置到右侧
                    if (scrollLines[i].xPosition + scrollLines[i].cachedTextWidth <= 0) {
                        scrollLines[i].xPosition = PANEL_RES_X;
                    }
                } else {
                    // 向右滚动：文本完全移出右侧时重置到左侧
                    if (scrollLines[i].xPosition >= PANEL_RES_X) {
                        scrollLines[i].xPosition = -scrollLines[i].cachedTextWidth;
                    }
                }
            }
        }
    }
}
```

### 渲染完整帧

```cpp
void DisplayManager::renderFrame() {
    if (!isDisplayReady()) {
        return;
    }

    // 清除后缓冲区
    dma_display->clearScreen();

    // 如果是全屏显示模式，绘制全屏内容
    if (isFullScreenDisplay && fullScreenContent) {
        dma_display->setTextColor(whiteColor);
        dma_display->setCursor(0, 0);
        dma_display->setTextWrap(true);
        dma_display->printlnUTF8(fullScreenContent);
    } else {
        // 否则绘制静态文本和滚动文本
        renderStaticTexts();
        renderScrollingTexts();
    }

    // 翻转DMA缓冲区
    dma_display->flipDMABuffer();
}
```

### 渲染滚动文本

```cpp
void DisplayManager::renderScrollingTexts() {
    if (!isDisplayReady() || scrollLines == nullptr) {
        return;
    }

    // 确保textWrap关闭（很重要！防止滚动文本被意外换行）
    dma_display->setTextWrap(false);

    for (int i = 0; i < maxLines; i++) {
        if (scrollLines[i].isActive && scrollLines[i].isScrolling && scrollLines[i].content) {
            // 如果文本完全超出屏幕，则跳过绘制
            int xPos = scrollLines[i].xPosition;
            int textWidth = scrollLines[i].cachedTextWidth;
            if (xPos >= PANEL_RES_X || xPos + textWidth <= 0) {
                continue;
            }

            // 先清除本行显示区域，避免旧静态文本或重叠内容干扰
            int lineHeight = getScrollTextHeight(i);
            if (lineHeight > 0) {
                dma_display->fillRect(0, scrollLines[i].yPosition, PANEL_RES_X, lineHeight, blackColor);
            }

            // 绘制滚动文本
            dma_display->setTextSize(scrollLines[i].textSize);
            dma_display->setTextColor(scrollLines[i].color);
            dma_display->setTextWrap(false);
            dma_display->setCursor(scrollLines[i].xPosition, scrollLines[i].yPosition);
            dma_display->printUTF8(scrollLines[i].content);
        }
    }

    // 恢复全局字体大小
    dma_display->setTextSize(textSize);
}
```

### 渲染静态文本

```cpp
void DisplayManager::renderStaticTexts() {
    if (!isDisplayReady() || staticTexts == nullptr) {
        return;
    }

    for (int i = 0; i < maxLines; i++) {
        if (staticTexts[i].isActive && staticTexts[i].content) {
            dma_display->setTextSize(staticTexts[i].textSize);
            dma_display->setTextColor(staticTexts[i].color);
            dma_display->setTextWrap(staticTexts[i].autoWrap);

            // 计算Y坐标
            int yPosition = (staticTexts[i].line - 1) * 16;
            dma_display->setCursor(0, yPosition);

            // 根据自动换行设置绘制文本
            if (staticTexts[i].autoWrap) {
                dma_display->printlnUTF8(staticTexts[i].content);
            } else {
                dma_display->printUTF8(staticTexts[i].content);
            }
        }
    }

    // 恢复全局字体大小
    dma_display->setTextSize(textSize);
}
```

### 渲染流程完整示例

#### 场景：静态文本 + 滚动文本

```
屏幕布局 (64x32):
┌─────────────────────────────────┐
│ Line 1: 静态标题 [font size=2]  │  ← 静态文本 (y=0-31)
├─────────────────────────────────┤
│ Line 2: 滚动通知 [font size=1]  │  ← 滚动文本 (y=16-31)
└─────────────────────────────────┘

渲染流程：
1. clearScreen() - 清除整个后缓冲区为黑色

2. renderStaticTexts()
   - 设置 font size=2
   - 绘制 "静态标题" 到 y=0
   - 占用区域: y=0~31 (font size=2 高度32)

3. renderScrollingTexts()
   - Line 2: 先清除区域 y=16~31 (黑色矩形)
   - 设置 font size=1
   - 绘制滚动文本到 y=16
   - 文本位置根据 xPosition 动态变化

4. flipDMABuffer()
   - 交换前后缓冲区
   - DMA开始显示新帧
```

#### 场景：多行滚动文本

```
屏幕布局 (64x32):
┌─────────────────────────────────┐
│ >>> Line 1滚动中 [size=1, left]  │  ← 滚动文本1
├─────────────────────────────────┤
│ <<< Line 2滚动中 [size=1, right] │  ← 滚动文本2
└─────────────────────────────────┘

渲染流程：
1. clearScreen() - 清除整个后缓冲区

2. renderStaticTexts() - 无静态文本，跳过

3. renderScrollingTexts()
   - Line 1: 
     - 清除区域 y=0~15
     - 更新 xPosition (向左移动1像素)
     - 绘制文本
   
   - Line 2:
     - 清除区域 y=16~31
     - 更新 xPosition (向右移动1像素)
     - 绘制文本

4. flipDMABuffer() - 交换缓冲区
```

### 性能优化要点

1. **帧率控制**
   - 50fps (frameInterval = 20ms)
   - 避免过度渲染浪费CPU

2. **可见性剔除**
   - 跳过完全超出屏幕的文本
   - 减少不必要的绘制操作

3. **缓存优化**
   - 缓存文本宽度避免重复计算
   - 减少getTextBounds()调用

4. **内存效率**
   - 动态分配文本内容
   - 及时释放不需要的文本

5. **DMA优势**
   - 硬件驱动显示，释放CPU
   - 双缓冲无闪烁
   - 刷新率稳定

## 注意事项

### 1. 行号范围

- 滚动文本：`line`必须在 `1` 到 `maxLines` 范围内
- 静态文本：`line <= 0`（包括-1）时清屏并全屏显示；`line > 0`时在指定行显示
- 超出范围的行号会被自动修正或忽略

### 2. 字体大小与覆盖

大字体会覆盖多行区域：

| 行号 | 字体1 | 字体2 |
|------|-------|-------|
| 1 | ✓ | ✓ |
| 2 | ✓ | ✓ |

- ✓表示该行被覆盖
- 系统会自动清除重叠的文本

### 3. 内存限制

- 每行文本动态分配内存，应避免过长的文本
- 系统会自动清理被覆盖的文本
- 使用`clearScrollLine()`或`clearAll()`手动清理

### 4. 屏幕分辨率适配

修改`PANEL_RES_Y`后，最大行数自动计算：

```cpp
// config.h
#define PANEL_RES_Y 32  // 64x32屏幕，支持2行
// 或
#define PANEL_RES_Y 64  // 64x64屏幕，支持4行
```

### 5. 性能优化

- 使用缓存的文本宽度（cachedTextWidth）避免重复计算
- 每帧只绘制可见区域的文本
- 跳过完全移出屏幕的文本

### 6. 静态文本与滚动文本混合

- 静态文本会自动清除重叠的滚动文本
- 滚动文本会自动清除重叠的静态文本
- 建议不要在相同区域混合显示

## 总结

多行滚动文本功能通过以下核心设计实现：

1. **动态最大行数**: `maxLines = PANEL_RES_Y / 16`
2. **独立字体记录**: 每行`ScrollLine`保存自己的`textSize`
3. **固定Y坐标系统**: `(line-1) * 16` 简化坐标计算
4. **动态字体切换**: 渲染时根据每行的`textSize`切换字体
5. **重叠检测机制**: 自动清除被覆盖的文本
6. **动态内存管理**: 使用`malloc/free`管理文本内容
7. **缓存优化**: 缓存文本宽度，避免重复计算
8. **高刷新率**: 50fps 渲染，DMA双缓冲

这些设计使得不同字体大小、速度、方向的文本可以互不干扰地同时滚动，并且能够自动适配不同的屏幕分辨率。
